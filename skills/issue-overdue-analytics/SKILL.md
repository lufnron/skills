# 问题列表超期分析统计

## 功能

读取Excel格式的问题列表，根据超期预警列内容自动分类统计：

1. **签收超期** - 包含"签收"关键词的超期问题（已超期和即将超期）
2. **处理超期** - 包含"处理超期"关键词的问题
3. **剩余处理时间** - 显示剩余处理时间的即将超期问题
4. **已处理待确认超期** - 已处理待确认阶段的超期问题
5. **按处理人统计** - 汇总每个处理人的超期问题数量和详细列表

## 使用方法

用户提供Excel问题列表文件后，运行分析脚本：

```bash
python scripts/analyze_issue_overdue.py 输入文件.xlsx 输出文件.xlsx
```

脚本自动：
1. **读取原始Excel问题列表**
2. **按超期类型分类筛选**到不同工作表
3. **按处理人统计汇总**超期数量
4. **输出文本汇总报告** - 直接打印完整统计和紧急问题列表，可复制发送
5. **输出分类汇总后的Excel文件** - 每个分类一个工作表

## 输入格式要求

输入Excel需要包含以下列：
- `问题标题` - 问题标题
- `所属项目` - 所属项目名称
- `项目经理` - 项目经理
- `问题处理组` - 问题处理组
- `提交人` - 提交人
- `处理人` - 当前处理人
- `问题状态` - 问题状态
- `超期预警` - 超期预警信息（包含超类型和时间）
- `应完成时间` - 应完成时间
- `问题链接` - 问题链接

## 输出格式

输出Excel包含多个工作表：
- `签收超期` - 所有签收相关超期问题，按紧急程度排序（已超期优先，时间越近越靠前）
- `处理超期` - 所有处理超期问题
- `剩余处理时间` - 剩余处理时间问题，按剩余天数从小到大排序
- `已处理待确认超期` - 已处理待确认超期问题
- `按处理人统计` - 每个处理人的超期问题数量统计

## 示例

用户："我会发你一个问题列表的excel，你帮我将问题签收、处理即将超期的全部筛选出来"

执行：
1. 保存用户上传的Excel文件到工作区
2. 运行 `python scripts/analyze_issue_overdue.py 用户上传.xlsx 问题超期分类汇总.xlsx`
3. 返回汇总结果，提供下载链接或上传到指定位置

## 排序规则

- **签收超期**：已超期排前面，按超期时长从长到短；即将超期排后面，按剩余时间从短到长
- **剩余处理时间**：按剩余天数从小到大排序，剩余天数越少越紧急
