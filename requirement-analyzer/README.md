# requirement-analyzer

> 需求文档处理与分析Skill
> 作者：刘丰荣
> 版本：v2.0.0

---

## 概述

本Skill提供完整的需求文档处理与分析能力，支持5步工作流和硬性门禁机制，聚焦于业务层面的需求分析。

## 功能特性

- 需求文档读取（支持Word/Markdown/TXT）
- 功能类型识别（纯定制/基于产品个性化）
- 业务故事线提取
- 页面字段分析（按页面拆分描述）
- 业务规则分析
- 特殊需求分析（批量操作、导入导出、消息通知、操作日志）
- 需求分析文档生成

## 快速开始

### 前置要求

- Python 3.7+
- python-docx 库

```bash
pip install python-docx
```

### 使用流程

1. **用户提供需求文档** - 支持Word、Markdown、TXT格式
2. **功能类型确认** - 识别纯定制或基于产品个性化
3. **需求分析** - 提取关键词、业务故事线、页面字段、业务规则
4. **不确定项确认** - 与用户确认模糊点
5. **结果确认** - 用户确认分析结果
6. **生成文档** - 输出结构化需求分析文档

## 核心工作流

```
步骤0: 功能类型确认【门禁】
    ↓
步骤1: 需求分析（关键词、业务故事线、页面字段）
    ↓
步骤2: 不确定项确认【门禁】
    ↓
步骤3: 结果确认【门禁】
    ↓
步骤4: 生成需求分析文档
```

## 输出文档结构

```markdown
## 1. 功能类型【必填】
## 2. 原始需求【必填】
## 3. 原始需求图片【如有】
## 4. 业务故事线【核心】
## 5. 功能清单【必填】（按页面拆分描述）
## 6. 不确定项及结论【如有】
```
