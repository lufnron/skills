## 前置要求

### Python环境

- Python 3.7+
- python-docx 库

**安装依赖**：
```bash
pip install python-docx
```

### 脚本调用方式

### Word文档读取（必须使用Python脚本）

**脚本路径**：`{Skill目录}/scripts/read_docx_improved.py`

**调用方式**：

```python
import subprocess
import json
import os

def read_word_document(file_path: str) -> dict:
    """
    读取Word文档
    @param {string} filePath - Word文档绝对路径
    @returns {Object} 解析结果
    """
    # 获取脚本路径（相对于当前工作目录）
    script_path = os.path.join(
        os.path.dirname(os.path.abspath(__file__)),
        '..', 'skills', 'requirement-analyzer', 'scripts', 'read_docx_improved.py'
    )
    script_path = os.path.normpath(script_path)

    command = f'python "{script_path}" "{file_path}"'

    result = subprocess.run(
        command,
        shell=True,
        capture_output=True,
        text=True,
        encoding='utf-8'
    )

    return json.loads(result.stdout)
```

**返回数据结构**：
```json
{
  "success": true,
  "paragraphs": ["段落1", "段落2"],
  "tables": [
    {
      "index": 1,
      "data": {
        "需求名称": "xxx",
        "需求说明及背景": "xxx"
      }
    }
  ],
  "images": ["image1.png"],
  "images_dir": "./temp_images"
}
```

### Markdown/TXT文档读取

直接使用Read工具读取文件内容即可。

### 错误处理

**常见错误及解决方案**：

| 错误类型 | 原因 | 解决方案 |
|----------|------|----------|
| python-docx未安装 | 缺少Python库 | 运行 `pip install python-docx` |
| Python命令找不到 | 系统未安装Python | 检查Python安装路径 |
| 文件路径不存在 | 文档路径错误 | 检查文件路径是否正确 |
| 编码错误 | 文件编码问题 | 尝试使用GBK编码 |

**备选方案**：
- 如果python-docx未安装，脚本会自动尝试安装
- 如果Python脚本仍然失败，检查系统Python环境配置

## 详细执行步骤

### 步骤0：功能类型确认【硬性门禁0】

**目标**：确定需求的功能类型

**功能类型定义**：

| 类型 | 说明 |
|------|------|
| 纯定制 | 全新功能，无产品基础 |
| 基于产品个性化 | 基于现有产品功能进行定制开发 |

**识别规则**：

1. **基于产品个性化关键词**：
   - 基于、扩展、修改、个性化、定制
   - 在...基础上、在...页面上
   - 产品模块、产品功能、产品源码

2. **纯定制关键词**：
   - 新建、全新、独立、从零开始
   - 重新开发、新建功能

**用户确认话术**：
```
**功能类型确认**

请确认功能类型：
- 纯定制（全新功能，无产品基础）
- 基于产品个性化（基于现有产品模块定制开发）

当前AI判断：[类型名称]（置信度：XX%）
```

**[步骤0完成]**

### 步骤1：需求分析

**目标**：深度分析需求内容，提取关键信息

#### 1.1 关键词提取

- 技术关键词：框架、数据库、接口等
- 业务关键词：业务术语、功能描述

#### 1.2 图片内容分析【重点】

**目标**：当需求文档包含图片时，必须结合图片内容进行分析

**图片来源**：
- Python脚本读取Word文档时，会自动提取图片到 `{文档名}_images` 目录
- 使用Read工具读取图片文件

**分析要求**：

1. **页面截图分析**
   - 分析页面布局、功能区域
   - 识别字段位置和排列方式
   - 识别按钮、链接等交互元素

2. **流程图分析**
   - 识别业务流程步骤
   - 理解状态流转逻辑
   - 识别分支条件和异常处理

3. **原型图分析**
   - 识别字段类型（下拉框、单选按钮等）
   - 识别表格列和查询条件
   - 识别弹窗和表单布局

4. **UI标注分析**
   - 识别字段的必填/可选
   - 识别输入格式要求
   - 识别校验规则提示

**图片分析格式**：

```
【图片分析：页面截图】

页面布局：
- 顶部：查询条件区域
- 中部：数据表格（列：名称、类型、状态）
- 底部：操作按钮（新增、编辑、删除）

识别字段：
- 名称：文本框，左侧
- 类型：下拉框，选项：企业/个体
- 状态：单选按钮组，是/否
- 创建时间：日期选择器
```

**注意**：
- 图片分析与文字描述相结合
- 如图片与文字描述有冲突，以图片为准
- 无法从图片识别的内容，列入不确定项

#### 1.3 业务故事线【重点】

**定义**：描述用户完成一个业务目标所经历的操作序列

**格式要求**：

```
【业务故事线 1：XXX功能】

角色：谁在操作
触发：什么情况下操作
步骤：
  1. [操作描述]
  2. [操作描述]
  3. [操作描述]
结果：得到什么/系统反馈什么

---（分隔不同故事线）
```

**示例**：

```
【业务故事线 1：新增场所信息】

角色：窗口业务员
触发：需要新增一个场所窗口信息
步骤：
  1. 点击"新增"按钮
  2. 填写场所基本信息（名称、地址、类型）
  3. 填写场所详细信息（负责人、联系电话）
  4. 上传场所资质图片
  5. 点击"保存"按钮
结果：系统提示"保存成功"，刷新列表显示新记录
```

**故事线数量**：每个核心功能至少提取1条业务故事线

#### 1.4 功能清单【按页面拆分描述】

**核心原则**：每个功能下的每个页面单独描述，不糅杂到一起

**格式要求**：

```
【功能名称：XXX功能】

### 1. 列表页面

#### 1.1 页面按钮
- 新增按钮
- 编辑按钮
- 删除按钮
- 导出按钮

#### 1.2 页面查询条件
| 字段名称 | 控件类型 | 说明 |
|----------|----------|------|
| 场所名称 | 文本框 | 模糊查询 |
| 场所分类 | 下拉框 | 选项：企业/个体/政府 |

#### 1.3 页面展示字段
| 字段名称 | 控件类型 | 说明 |
|----------|----------|------|
| 场所名称 | 文本 | - |
| 场所分类 | 文本 | - |
| 详细地址 | 文本 | - |
| 状态 | 标签 | 草稿/在用 |

#### 1.4 业务规则
- 场所名称最大50字符，不能重复
- 删除仅草稿状态可操作

#### 1.5 特殊需求
- 导出Excel：是

---

### 2. 新增/编辑弹窗

#### 2.1 页面按钮
- 保存按钮
- 取消按钮

#### 2.2 表单字段
| 字段名称 | 控件类型 | 必填 | 说明 |
|----------|----------|------|------|
| 场所名称 | 文本框 | 是 | 最大50字符 |
| 场所分类 | 下拉框 | 是 | 选项：企业/个体/政府 |
| 区划名称 | 下拉框 | 是 | 树选择 |
| 街道名称 | 下拉框 | 是 | 级联选择 |
| 统一对外服务时间 | 时间选择器 | 否 | - |
| 咨询电话 | 文本框 | 否 | 11位手机号 |
| 投诉电话 | 文本框 | 否 | 11位手机号 |
| 是否社区办理 | 单选按钮组 | 是 | 是/否 |
| 状态 | 单选按钮组 | 是 | 草稿/在用 |

#### 2.3 业务规则
- 详细地址 = 区划名称 + 街道名称 + 路/弄/号（自动拼接）
- 场所名称同一类型下不能重复

#### 2.4 特殊需求
- 版本管理：修改后版本+1，历史保留

---

【功能名称：XXX功能2】

...（同上格式）
```

**格式说明**：

1. **功能名称**：每个功能单独一个章节
2. **页面拆分**：列表页面、新增/编辑弹窗等单独描述
3. **页面按钮**：页面的操作按钮
4. **页面查询条件**：列表页的查询过滤条件
5. **页面展示字段/表单字段**：字段的详细描述
6. **业务规则**：该页面的约束条件
7. **特殊需求**：导入导出、消息通知、操作日志等

**注意**：
- 不要把所有页面字段都堆在一个表格里
- 列表页面和表单页面分开描述
- 按钮、查询条件、展示字段分开描述
- 字段需要标注控件类型（文本框/下拉框/单选按钮组等）

**字段类型分类**（参考示例，非固定列表）：

| 字段类型 | 说明 | 示例 |
|----------|------|------|
| 文本框 | 单行文本输入 | 名称、地址 |
| 下拉框 | 下拉选择 | 分类、状态 |
| 单选按钮组 | 单项选择 | 是/否、性别 |
| 复选框 | 多项选择 | 权限、标签 |
| 日期选择器 | 日期输入 | 创建时间 |
| 上传控件 | 文件/图片上传 | 附件、头像 |
| 文本域 | 多行文本 | 备注、描述 |
| 数字输入 | 数值输入 | 数量、金额 |
| 树选择 | 树形结构选择 | 组织架构、分类 |

> **注意**：以上为参考示例，非固定列表。AI应灵活识别，未知类型按语义归类或在不确定项中确认。

**示例**：

```
【功能：场所信息管理】

页面字段分析：
列表页：
- 表格列：场所名称、场所类型、负责人、联系电话、状态

新增/编辑弹窗：
- 场所名称：文本框，必填，最大50字符
- 场所类型：下拉框，必填，选项：企业/个体/政府
- 负责人：文本框，必填
- 联系电话：文本框，可选，11位手机号
- 经营状态：单选按钮组，必填，营业中/已停业
- 场所图片：上传控件，可选，支持jpg/png
- 备注：文本域，可选，最大500字符

业务规则/约束条件：
- 场所名称：最大50字符，同一区域下不能重复
- 联系电话：11位手机号或固定电话（带区号）
- 场所类型：选择后关联不同的资质要求

批量操作：
- 批量删除：是
- 批量审核：是

导入导出：
- 导入：是，Excel模板
- 导出：是，Excel格式

消息通知：
- 站内信：是，新增成功通知管理员
- 短信：否

操作日志：
- 记录：是
- 记录范围：新增、修改、删除
```

**字段分析要点**：
1. 区分列表页字段和表单字段
2. 明确字段的输入方式（文本框/下拉框/单选等）
3. 标注必填/可选
4. 标注验证规则（最大长度、格式等）
5. 特殊控件（上传、日期、树选择等）

## 常见问题（FAQ）

### Q1：如何判断功能类型？

A：参考步骤0中的关键词识别规则，优先判断基于产品个性化，如果无法判断则让用户确认。

### Q2：用户不配合确认怎么办？

A：硬性门禁机制下，必须等待用户确认。如果用户持续不配合，可以提示："需求分析需要您的确认才能继续，请问是否有疑问？"

### Q3：需求文档包含图片怎么办？

A：
1. Python脚本读取Word文档时，会自动提取图片到 `{文档名}_images` 目录
2. 使用Read工具读取图片内容
3. 必须结合图片进行分析：识别页面布局、字段类型、交互元素等
4. 如图片与文字描述有冲突，以图片为准
5. 将图片分析结果融入功能清单和业务故事线中

### Q4：Python脚本调用失败怎么办？

A：
1. 检查Python是否已安装（运行 `python --version`）
2. 检查python-docx库是否已安装（运行 `pip show python-docx`）
3. 如果都正常，尝试使用MCP的read_document工具作为备选

### Q5：如何处理未知字段类型？

A：
1. 根据语义推断最接近的字段类型
2. 如果无法判断，记录为"文本框"
3. 在不确定项中向用户确认具体控件类型

### Q6：业务规则如何提取？

A：
1. 从需求描述中识别约束条件
2. 从业务场景中推断规则
3. 从常见规则中补充
